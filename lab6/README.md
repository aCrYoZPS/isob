# Лабораторная работа: Защита от SQL-Injection

## Описание
Данный проект демонстрирует разницу между уязвимым и защищенным кодом при работе с SQL-базой данных в Python (SQLite).

## Файлы
- `sql_injection_demo.py`: Основной скрипт с демонстрацией.

## Запуск
```bash
python3 sql_injection_demo.py
```

## Как это работает
1. **Уязвимая функция (`vulnerable_get_user`)**:
   - Использует: `f"SELECT * FROM users WHERE username = '{username}'"`
   - Опасность: Пользователь может ввести `' OR '1'='1`, изменив логику SQL-запроса.

2. **Защищенная функция (`secure_get_user`)**:
   - Использует: `cursor.execute("SELECT * FROM users WHERE username = ?", (username,))`
   - Защита: Драйвер БД безопасно подставляет параметры, исключая возможность выполнения внедренного кода.
